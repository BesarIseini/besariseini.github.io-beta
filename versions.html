Beta 7 — Text, Numeric & Tolerance Systems 09/02/2026

✅ Added

  • New question types:
    • Short Text
    • Long Text
    • Numeric
    • Fill-in-the-Blanks
  • String tolerance system for text-based questions
  • Number tolerance system with multiple evaluation modes
  • Reworked the question warning system for improved flexibility, styling control, and extensibility

📝 Short Text Type

  • Single-word text answer
  • Consists of question, description, and input field
  • Includes a per-question string tolerance system (default = 1)
  • Input is restricted to one word (spaces are automatically removed, even when pasted)
  • Review feedback supports four outcomes:
    • Perfect Answer
    • Small Typo Answer (-20%)
    • Your answer was wrong
    • You haven’t answered
  • Visual checkmark indicator:
    • Green = fully correct
    • Yellow = small typo (within tolerance)
    • Red = incorrect or missing

⚠️ Short Text Known Issues

  • Typo penalty is currently fixed at 80% (not configurable)
  • Does not account for synonyms or alternative correct answers

📝 Long Text Type

  • Supports multiple words
  • Consists of question, description, and input field

⚠️ Long Text Known Issues

  • Answers are manually reviewed (no automatic grading)

🔢 Numeric Type

  • Input restricted to numeric values only
  • Consists of question, description, and input field
  • Includes a per-question numeric tolerance system
  • Visual checkmark indicator:
    • Green = correct or within tolerance
    • Red = incorrect or missing

🧩 Fill-in-the-Blanks Type

  • Consists of multiple Short Text inputs (numeric support planned)
  • Each input has its own visual correctness indicator

⚠️ Fill-in-the-Blanks Known Issues

  • Numeric values are currently treated as strings

🔤 String Tolerance System

  • Character swap counts as 1 difference
  • Extra or repeated character counts as 1 difference
  • Missing character counts as 1 difference
  • If total differences ≤ tolerance → 80% score
  • If total differences = 0 → 100% score

🔢 Number Tolerance System

  • Supported modes:
    • Absolute — fixed ± range around correct answer
    • Percentage — ± percentage range
    • Range — explicit min/max bounds
    • Strict — exact match only
  • Evaluation returns:
    • 0 → within tolerance
    • 1 → outside tolerance
  • Recommendations:
    • Absolute → floats
    • Percentage → large numbers
    • Range → explicit control
    • Strict → integers

⚠️ Known Quiz Issues

  • String and Number tolerance systems are currently separate, causing awkward behavior in Fill-in-the-Blanks questions

⚠️ Question Warning Rework

  • Question warning rendering is now a separate core system from quiz rendering
  • Multiple warnings can now be rendered per question
  • Added new warning format: CUSTOM
  • To apply custom styles, the warning format must be set to CUSTOM
  • NONE no longer applies any CSS class
  • CUSTOM bypasses NONE unless overridden
  • Warning styles are now defined via optional structured properties

🛠️ Technical

  • Warning styling properties are grouped into optional sub-objects:
    • Text
    • Layout
    • Flex
    • Visual
    • Border
    • Positioning
  • New optional styling properties include:
    • Border (width, color, style, radius)
    • Padding and margin
    • Text alignment and overflow control
    • Positioning (top, left, right, bottom, z-index)
    • Flex behavior
    • Transform, opacity, visibility, display, pointer events
  • Renamed question types:
    • Short_Answer → Short_Text
    • Long_Answer → Long_Text
